Getting Started

Add Forge.Platform to a new ASP.NET Core application, define your first entity, and query it — in under ten minutes.

Prerequisites

Runtime
net10.0 — .NET 10 SDK
IDE
Visual Studio 2022 17.12+, Rider 2024.3+, or VS Code with C# Dev Kit
Source generators
Roslyn source generators are consumed automatically via the OutputItemType="Analyzer" project reference; no manual step required.

Step-by-step setup

  1. Add packages

    Reference the core slices you need. For a minimal entity-backed REST API:

    PackageRole
    Forge.EntityEntity attributes, IEntity, EntityBase
    Forge.Entity.GeneratorsRoslyn generator — structural partial class, IRI sealing
    Forge.OperationsEntityOperations static CRUD helpers, [OperationEndpoints]
    Forge.Operations.GeneratorsRoslyn generator — CreateAsync, ReadAsync, UpdateAsync, DeleteAsync, ListAsync
    Forge.RepositoryIEntityStore, ITransactionalEntityStore, EntityTransaction
    Forge.Repository.InMemoryIn-memory dotNetRDF backend (no infrastructure required)
    Forge.Execution.HttpCorrelation threading, BranchScope middleware, HTTP helpers
    Forge.Operations.HttpMapOperations() — five REST endpoints per entity
     
    dotnet add package Forge.Entity
dotnet add package Forge.Entity.Generators
dotnet add package Forge.Operations
dotnet add package Forge.Operations.Generators
dotnet add package Forge.Repository
dotnet add package Forge.Repository.InMemory
dotnet add package Forge.Execution.Http
dotnet add package Forge.Operations.Http
  2. Wire DI in Program.cs

    Register the platform services in dependency-injection order. Below is the minimal wiring derived from samples/Application.Sample/Program.cs:

    var builder = WebApplication.CreateBuilder(args);

// 1. Entity repository — InMemory backend for development
var repoBuilder = builder.Services.AddForgeEntityRepository(builder.Configuration);
repoBuilder.UseInMemory();

// 2. HTTP transport for entity operations
//    Scans MyAssembly for [Entity]+[OperationEndpoints] types
builder.Services.AddOperationEndpointsHttpFromAssemblyContaining<Book>();

var app = builder.Build();

// 3. Map five REST endpoints per [OperationEndpoints] entity
app.MapOperations();

app.Run();
    Tip: To switch to GraphDB, call repoBuilder.UseGraphDb() instead of UseInMemory() and set Forge__EntityRepository__Backend=GraphDb plus the Forge:GraphDb:* connection settings.
  3. Define your first entity

    Annotated partial classes are the entity model. The source generator emits the structural half at compile time:

    using Forge.Entity;
using Forge.Operations;

// [Entity] declares the RDF type and URL path segment.
// [Identity] controls how the IRI suffix is generated.
// [OperationEndpoints] tells MapOperations() to register REST routes.
[Entity(Path = "books")]
[Identity(IdentityGenerator.PropertyBasedPlain)]
[OperationEndpoints]
public partial class Book
{
    // [IdentityPart] marks the property whose value forms the IRI suffix.
    [IdentityPart(0)]
    [Predicate("isbn")]
    public partial string Isbn { get; init; }

    // Plain predicates are stored as RDF literals.
    [Predicate("title")]
    public string Title { get; set; } = string.Empty;

    [Predicate("author")]
    public string Author { get; set; } = string.Empty;

    [Predicate("publishedYear")]
    public int PublishedYear { get; set; }

    [Predicate("available")]
    public bool Available { get; set; } = true;
}
    What the generator emits: Forge.Entity.Generators emits a partial class that implements IEntity, seals Iri from the identity properties, and wires EntityRef<T> / EntityRefCollection<T> backing fields for relation properties. Forge.Operations.Generators emits static CreateAsync, ReadAsync, UpdateAsync, DeleteAsync, and ListAsync methods on the partial class.
  4. Register the entity type

    The assembly scan performed by AddOperationEndpointsHttpFromAssemblyContaining<T>() automatically calls RegisterEntity<T>() for every [OperationEndpoints] type it discovers. For types you want to register manually you can also call:

    repoBuilder.RegisterEntity<Book>();
  5. Save and query an entity

    Save via EntityTransaction (the transactional write unit), read via generated helpers or IEntityRepository<T>:

    // ── Save ──────────────────────────────────────────────────────────────────
// Inject ITransactionalEntityStore or use the generated CreateAsync helper.
// Using the generated helper (requires EntityOperations.Use() scope):
await using var tx = EntityOperations.BeginTransaction();
tx.Create(new Book
{
    Isbn = "978-3-16-148410-0",
    Title = "Domain-Driven Design",
    Author = "Eric Evans",
    PublishedYear = 2003,
    Available = true
});
await tx.CommitAsync();

// ── Query all ─────────────────────────────────────────────────────────────
// Via generated ListAsync (inside an EntityOperations.Use scope):
await foreach (var book in Book.ListAsync())
{
    Console.WriteLine($"{book.Isbn}: {book.Title}");
}

// ── Query by IRI ─────────────────────────────────────────────────────────
var iri = "https://forge-it.net/books/978-3-16-148410-0";
Book? found = await Book.ReadAsync(iri);
    Branch scope: Wrap reads and writes in BranchScope.Use(branchIri) to target a named-graph branch rather than the default main graph.
