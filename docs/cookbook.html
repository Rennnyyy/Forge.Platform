Cookbook

Copy-paste patterns for the most common Forge.Platform scenarios.

Entity

1 · Define a simple entity

When you need a new domain concept persisted as RDF triples with auto-generated REST endpoints.

using Forge.Entity;
using Forge.Operations;

[Entity(Path = "artists")]               // RDF type IRI segment + URL path
[Identity(IdentityGenerator.Random)]     // UUID-based IRI suffix
[OperationEndpoints]                     // registers POST/GET/PUT/DELETE via MapOperations()
public partial class Artist
{
    [Predicate("name")]
    public string Name { get; set; } = string.Empty;

    [Predicate("country")]
    public string Country { get; set; } = string.Empty;

    [Predicate("bio")]
    public string? Bio { get; set; }

    [Predicate("active")]
    public bool Active { get; set; } = true;

    [Predicate("debutYear")]
    public int DebutYear { get; set; }
}

2 · Define a related entity (object reference)

When an entity owns a reference to another entity (N:1 or M:N).

using Forge.Entity;
using Forge.Operations;

[Entity(Path = "studios", PredicatePath = "studio")]
[Identity(IdentityGenerator.Random)]
[OperationEndpoints]
public partial class Studio
{
    [Predicate("name")]
    public string Name { get; set; } = string.Empty;

    // N:1 — stored as a single IRI predicate in this entity's RDF graph.
    // The generator emits an EntityRef<Artist> backing field.
    [Predicate("managedBy")]
    public partial EntityRef<Artist>? ManagedBy { get; set; }

    // 1:N ordered list — stored as rdf:List of Recording IRIs.
    [Predicate("hasRecording")]
    public partial EntityRefCollection<Recording> Recordings { get; }

    // M:N — same Genre IRI can appear in many studios.
    [Predicate("hasGenre")]
    public partial EntityRefCollection<Genre> Genres { get; }
}

// Inverse side — read-only, populated at load time by the inverse-ref loader.
// Declare on the referenced type:
// [Inverse(nameof(Studio.ManagedBy), "managedBy")]
// public partial EntityRefCollection<Studio> ManagedStudios { get; }

3 · HTTP CRUD endpoints for an entity

When you want five REST endpoints (Create / List / Read / Update / Delete) auto-registered from an entity class.

// In Program.cs:

// Registers one OperationEndpointDescriptor per [OperationEndpoints] type found
// in the assembly that contains Book.
builder.Services.AddOperationEndpointsHttpFromAssemblyContaining<Book>();

// ...

// Wires the five endpoints under api/entities/{path}:
//   POST   api/entities/books         — Create
//   GET    api/entities/books         — List
//   GET    api/entities/books?iri=…   — Read single
//   PUT    api/entities/books?iri=…   — Update
//   DELETE api/entities/books?iri=…   — Delete
app.MapOperations();

Branching

4 · Scope a request to a named branch

When you want all entity reads and writes within a request to target a specific named-graph branch rather than the default main graph.

// In Program.cs — register + activate the middleware:
builder.Services.AddBranchHttp(builder.Configuration); // reads/writes X-Forge-BranchIri
app.UseBranchScope();                                  // sets BranchScope.Current per request

// From any client — pass the branch IRI in the request header:
//   X-Forge-BranchIri: https://forge-it.net/branches/my-feature-branch
//
// The response echoes the effective IRI:
//   X-Forge-Effective-BranchIri: https://forge-it.net/branches/my-feature-branch

// From code — wrap in BranchScope.Use() (ambient scope, no DI required):
using (BranchScope.Use("https://forge-it.net/branches/my-feature-branch"))
{
    await foreach (var book in Book.ListAsync())
    {
        // books read from the named graph for that branch
    }
}

Capabilities

5 · Implement a synchronous capability

When you need a typed command/response handler with aspect validation and correlated execution.

using Forge.Capability;

// Command and response are plain classes.
public sealed record CreateItemCommand(string Name, decimal Price);
public sealed record CreateItemResponse(string Iri);

// Handler — implement ICapabilityHandler<TCommand, TResponse>.
// [Capability] declares the HTTP route and optionally the messaging topic.
[Capability(Group = "catalog", Name = "items/create", Method = HttpMethod.Post)]
public sealed class CreateItemHandler
    : ICapabilityHandler<CreateItemCommand, CreateItemResponse>
{
    public async ValueTask<ExecutionResult<CreateItemResponse>> HandleAsync(
        CreateItemCommand command,
        CapabilityContext context,
        CancellationToken cancellationToken)
    {
        // validate, create entity, etc.
        var iri = $"https://forge-it.net/items/{Guid.NewGuid()}";
        return ExecutionResult.Ok(new CreateItemResponse(iri));
    }
}

// Registration in Program.cs:
builder.Services.AddCapabilityHandlersFromAssemblyContaining<CreateItemHandler>();
builder.Services.AddCapabilityHttp();
// ...
app.MapCapabilities();
// → POST api/capabilities/catalog/items/create

6 · Dispatch a capability over messaging (async)

When you need fire-and-forget or request-reply dispatch via a message broker (ADR-0022).

// Registration in Program.cs:
builder.Services.AddForgeMessagingInMemory();           // or AddForgeMessagingKafka()
builder.Services.AddForgeCapabilityMessaging();         // auto-discovers [Capability] handlers

// Fire-and-forget:
app.MapPost("/api/fire", async (
    MyCommand cmd,
    IAsyncCapabilityDispatcher<MyCommand, MyResponse> dispatcher,
    CancellationToken ct) =>
{
    await dispatcher.PublishAsync(cmd, cancellationToken: ct);
    return Results.Accepted();
});

// Request-reply — awaits the ExecutionResult from the handler:
app.MapPost("/api/dispatch", async (
    MyCommand cmd,
    IAsyncCapabilityDispatcher<MyCommand, MyResponse> dispatcher,
    CancellationToken ct) =>
{
    var result = await dispatcher.PublishAndWaitAsync(cmd, cancellationToken: ct);
    return result switch
    {
        ExecutionResult<MyResponse>.Ok ok     => Results.Ok(ok.Response),
        ExecutionResult<MyResponse>.Fail fail => Results.UnprocessableEntity(fail.Error),
        _                                      => Results.StatusCode(500)
    };
});

Messaging

7 · Emit entity change events

When you want every entity write to be published as an event to a history topic and a compacted state topic (ADR-0021).

// In Program.cs — order matters: messaging before aspects:
builder.Services.AddForgeMessagingInMemory();   // or Kafka
builder.Services.AddForgeEntityEvents();        // wires EventEmittingEntityStore decorator

// Register per-entity messaging (serializer + topic names):
builder.Services.AddForgeEntityMessaging<Book>(opts =>
{
    opts.TypeIri = "https://forge-it.net/entities/Book";
    // Topics auto-derived: forge.entities.book.history  (append-only)
    //                      forge.entities.book.state    (compacted latest state)
});

// Consume events — implement a hosted service or background worker:
public class BookHistoryConsumer : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        // Inject IMessageConsumer<string, EntityChangedEnvelope<BookDto>>
        await foreach (var envelope in _consumer.ConsumeAsync(
            "forge.entities.book.history", stoppingToken))
        {
            // envelope.Payload.Operation: Created | Updated | Deleted
            Console.WriteLine($"[{envelope.Payload.Operation}] {envelope.Payload.Iri}");
        }
    }
}

Storage

8 · Object-bearing entity (file upload / download)

When an entity is primarily a metadata wrapper for a binary blob (ADR-0023).

using Forge.Entity;
using Forge.ObjectStorage;
using Forge.Operations;

// Mark entity with [ObjectBearing] — MapOperations() skips it;
// MapObjectOperations() registers eight routes instead.
[Entity(Path = "track-masters", PredicatePath = "track-master")]
[Identity(IdentityGenerator.Random)]
[ObjectBearing]
[OperationEndpoints]
public partial class TrackMaster
{
    [Predicate("title")]
    public string Title { get; set; } = string.Empty;

    [Predicate("durationSeconds")]
    public int DurationSeconds { get; set; }
}

// In Program.cs:
builder.Services.AddForgeObjectStorageInMemory();
builder.Services.AddForgeObjectStorageHttpFromAssemblyContaining<TrackMaster>();
// ...
app.MapObjectOperations();

// Registered routes for TrackMaster:
//   POST   api/entities/track-masters              — Create metadata
//   GET    api/entities/track-masters              — List metadata
//   GET    api/entities/track-masters?iri=…        — Read metadata
//   PUT    api/entities/track-masters?iri=…        — Update metadata
//   DELETE api/entities/track-masters?iri=…        — Delete entity + blob
//   PUT    api/objects/track-masters/content?iri=… — Upload binary asset
//   GET    api/objects/track-masters/content?iri=… — Download binary asset
//   DELETE api/objects/track-masters/content?iri=… — Delete blob only

Security

9 · Add a SHACL aspect guard

When you need write operations to be validated against a SHACL shape before they are committed.

// Register the aspect infrastructure (enables AspectEnforcingTransactionalStore):
builder.Services.AddForgeAspects();

// Register a SHACL operation aspect from inline Turtle:
builder.Services.AddOperationAspect(sp => new InlineTtlOperationAspect(
    iri: "urn:forge:aspects:book-v1",
    shapeTtl: """
        @prefix sh: <http://www.w3.org/ns/shacl#> .
        @prefix ex: <https://forge-it.net/predicates/book/> .

        [] a sh:NodeShape ;
           sh:targetClass <https://forge-it.net/entities/Book> ;
           sh:property [
               sh:path ex:isbn ;
               sh:minLength 13 ;
               sh:maxLength 13 ;
               sh:message "ISBN must be exactly 13 characters." ] .
    """));

// Activate on a write request by sending the header:
//   X-Forge-Operation-AspectIri: urn:forge:aspects:book-v1
//
// Writes that violate the shape return 422 Unprocessable Entity with
// the SHACL violation messages in the response body.

10 · Apply an authorization rule

When you need every write to pass through an authorization guard before the aspect or backend layer.

// Implement IAspectGuard:
public sealed class MyAuthGuard : IAspectGuard
{
    public ValueTask<bool> IsAllowedAsync(
        string agentToken,
        TransactionOperation operation,
        CancellationToken ct)
    {
        // agentToken comes from AuthorizationContext.CurrentAgentToken
        // (set by X-Forge-Agent-Token header via UseAgentTokenMiddleware)
        return ValueTask.FromResult(agentToken == "trusted-agent");
    }
}

// Register — wraps ITransactionalEntityStore with GuardedTransactionalStore:
builder.Services.AddForgeAuthorizationHttp(builder.Configuration);
// Also register your guard:
builder.Services.AddSingleton<IAspectGuard, MyAuthGuard>();
// Activate middleware:
app.UseAgentTokenMiddleware();
