Configuration Reference

Every AddForge…(), UseForge…(), and Map…() registration method with its parameters and what it wires up.

Note: All registrations follow the managed-entity keyed store pattern described in ADR-0019. Registration order is irrelevant thanks to the deferred-factory decorator chain described in ADR-0014.

Forge.Repository

MethodParametersRegisters / configures
AddForgeEntityRepository(config) IConfiguration configuration Binds EntityRepositoryOptions from the Forge:EntityRepository section. Returns a ForgeEntityRepositoryBuilder for chaining backend and entity registration calls.
RegisterEntity<T>() (on builder) none Registers IRdfMapper<T>, IEntityRepository<T>, and all generated mappers for entity type T. 
// EntityRepositoryOptions defaults
new EntityRepositoryOptions
{
    Backend          = "InMemory",   // "InMemory" | "GraphDb"
    DefaultBranchIri = "",           // seeded by AddForgeBranchHttp()
    NamedGraph       = null,         // null = use branch-scoped named graph
    TypeBaseIri      = null          // null = derived from EntityOptions.BaseIri
};

Forge.Repository.InMemory

MethodParametersRegisters / configures
UseInMemory() (on builder) none Registers an in-memory dotNetRDF store as the BackendStoreKey keyed ITransactionalEntityStore. No external infrastructure required.

Forge.Repository.GraphDb

MethodParametersRegisters / configures
UseGraphDb() (on builder) none Registers an Ontotext GraphDB HTTP store as the backend. Reads connection settings from Forge:GraphDb:BaseUrl and Forge:GraphDb:RepositoryId. 
// GraphDb configuration keys in appsettings.json:
{
  "Forge": {
    "EntityRepository": { "Backend": "GraphDb" },
    "GraphDb": {
      "BaseUrl":       "http://localhost:7200",
      "RepositoryId":  "forge-sample"
    }
  }
}

Forge.Aspects

MethodParametersRegisters / configures
AddForgeAspects() none Registers IAspectStore and decorates the unkeyed ITransactionalEntityStore with AspectEnforcingTransactionalStore. Also registers ManagedEntityAspectValidationService (startup validator, conditional on IAspectStore presence).
AddForgeAspectsForKeyedStore(storeKey, sparqlResolver) string storeKey, Func<IServiceProvider, ISparqlQueryStore> sparqlStoreResolver Decorates the named keyed store with AspectEnforcingTransactionalStore. Required for managed-entity HTTP layers (ADR-0019). Registers a matching AspectEnforcedKeyedStoreRegistration(storeKey).
AddOperationAspect(factory) Func<IServiceProvider, IOperationAspect> factory Registers an IOperationAspect in IAspectStore. Activated on write requests by header X-Forge-Operation-AspectIri.
AddMessageAspect(factory) Func<IServiceProvider, IMessageAspect> factory Registers an IMessageAspect in IAspectStore. Activated on capability dispatch requests by header X-Forge-Capability-AspectIri.
AddQueryAspect(factory) Func<IServiceProvider, IQueryAspect> factory Registers an IQueryAspect in IAspectStore, governing SPARQL query filtering.

Forge.Authorization + Forge.Authorization.Http

MethodParametersRegisters / configures
AddForgeAuthorization() none Registers IAspectGuard and decorates ITransactionalEntityStore with GuardedTransactionalStore.
AddForgeAuthorizationHttp(config) IConfiguration configuration Registers GuardedTransactionalStore decorator, binds AuthorizationOptions from Forge:Authorization section, and registers a startup filter that enforces RequireExplicitGuard.
UseAgentTokenMiddleware() none Reads X-Forge-Agent-Token request header and sets AuthorizationContext.CurrentAgentToken for the request lifetime. 
// AuthorizationOptions defaults (Forge:Authorization section):
{
  "Forge": {
    "Authorization": {
      "RequireExplicitGuard": true   // false = uses AllowAllAspectGuard (dev only)
    }
  }
}

Forge.Branch + Forge.Branch.Http

MethodParametersRegisters / configures
AddForgeBranch(config) IConfiguration configuration Registers the management-graph store (keyed "forge.branch.management" and "forge.branch.management.raw"), sets BranchDefault.BranchIri from Forge:Branch:DefaultBranchIri, and starts BranchSeedingService to upsert the default branch entity on first boot.
AddForgeBranchHttp(config) IConfiguration configuration Calls AddForgeBranch() and wires AspectEnforcingTransactionalStore onto the management store via AddForgeAspectsForKeyedStore(). Registers HeaderExecutionAspectIriProvider for the management surface. Required for the aspect-IRI threading obligation of ADR-0019.
AddBranchHttp(config) IConfiguration configuration Registers HeaderBranchIriProvider (reads X-Forge-BranchIri) and binds BranchOptions. Does not register the management store.
UseBranchScope() none Middleware: reads X-Forge-BranchIri, calls BranchScope.Use(iri), echoes the effective IRI in X-Forge-Effective-BranchIri response header.
MapBranches() none Registers five REST endpoints for Branch entity CRUD under api/branches.
MapSnapshots() none Registers POST and DELETE endpoints for snapshot management under api/snapshots.

Forge.Capability + Forge.Capability.Http + Forge.Capability.Messaging

MethodParametersRegisters / configures
AddCapabilityHandler<TCmd,TResp,THandler>() none Registers a single ICapabilityHandler and its metadata.
AddCapabilityHandlersFromAssemblyContaining<T>() none Scans the assembly containing T for all types implementing ICapabilityHandler<,> and registers them with their [Capability] metadata. Must be called before AddCapabilityHttp().
AddCapabilityHttp() none Builds endpoint metadata from registered capability handlers. Must be called after AddCapabilityHandlers…().
MapCapabilities() none Registers one HTTP endpoint per [Capability] handler under api/capabilities/{group}/{name}.
AddForgeCapabilityMessaging() none Auto-discovery variant — scans registered capability handlers and derives topic names (forge.capabilities.{identity}.commands / .replies), wires IAsyncCapabilityDispatcher, command-pump, and reply listener for each.
AddForgeCapabilityMessaging<TCmd,TResp>(options) Action<CapabilityMessagingOptions>? configure Producer-side: wires IAsyncCapabilityDispatcher<TCmd,TResp> for a specific command/response pair.
AddForgeCapabilityConsumer<TCmd,TResp>(options) Action<CapabilityMessagingOptions>? configure Consumer-side: wires ICapabilityMessageConsumer<TCmd,TResp> and the reply publisher for a specific command/response pair.

Forge.Messaging.InMemory + Forge.Messaging.Kafka + Forge.Entity.Messaging

MethodParametersRegisters / configures
AddForgeMessagingInMemory() none Registers InMemoryMessageBroker as singleton IMessageProducer and IMessageConsumer for all key/value types. No external broker required.
AddForgeMessagingKafka(options) Action<KafkaOptions>? configure = null Registers Confluent.Kafka-backed IMessageProducer and IMessageConsumer. Reads bootstrap servers and SASL settings from Forge:Kafka:*.
AddForgeEntityEvents() none Decorates the decorated ITransactionalEntityStore with EventEmittingEntityStore. Must be called before AddForgeAspects() / AddForgeAuthorizationHttp() so the event decorator sits above the aspect layer in the chain.
AddForgeEntityMessaging<TEntity>(options) Action<EntityMessagingOptions>? configure = null Registers the IMessageSerializer and topic configuration for entity type TEntity. Options: TypeIri, HistoryTopic (default: forge.entities.{type}.history), StateTopic (default: forge.entities.{type}.state).

Forge.ObjectStorage.InMemory + Forge.ObjectStorage.Http

MethodParametersRegisters / configures
AddForgeObjectStorageInMemory() none Registers an in-memory IObjectStoreProvider that creates a keyed IObjectStore per entity type's StoreKey.
AddForgeObjectStorageHttp(descriptor) ObjectStorageEndpointDescriptor descriptor Registers a single [ObjectBearing] entity type for eight-route HTTP wiring.
AddForgeObjectStorageHttpFromAssemblyContaining<T>() none Scans the assembly containing T for all [ObjectBearing] entity types and registers their ObjectStorageEndpointDescriptors.
MapObjectOperations() none Registers eight REST routes per [ObjectBearing] entity: metadata CRUD (5 routes), plus Upload, Download, and Blob-Delete (3 routes).

Forge.Operations.Http

MethodParametersRegisters / configures
AddOperationEndpointsHttp(descriptor) OperationEndpointDescriptor descriptor Registers one OperationEndpointDescriptor for a single entity type, enabling MapOperations() to wire the five REST endpoints for it.
AddOperationEndpointsHttpFromAssemblyContaining<T>() none Scans the assembly containing T for all [OperationEndpoints] entity types (excluding [ObjectBearing]) and registers their descriptors.
MapOperations() none Wires five REST endpoints per discovered entity: POST (Create), GET (List), GET?iri= (Read), PUT?iri= (Update), DELETE?iri= (Delete) under api/entities/{path}.
