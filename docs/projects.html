Forge.Platform — Projects

Forge.Platform is a .NET library platform for building richly-typed, RDF-aware domain models. The projects below deliver a complete stack: compile-time code generation via Roslyn source generators, IRI-based entity identity and an abstract RDF persistence contract, a LINQ-to-SPARQL query translation layer, concrete store backends (in-memory and Ontotext GraphDB), and an ambient active-record operations bridge — all without runtime reflection.

Forge.Entity

Target framework
net10.0
Namespace
Forge.Entity

The core runtime library. Defines the entity model (EntityBase, IEntity), the full annotation surface ([Entity], [Identity], [IdentityPart], [Owning], [Inverse], [Required], [Enumeration]), identity strategies and IRI materialization, and the lazy-loading wiring types (EntityRef<T>, EntityRefCollection<T>, EntitySession). Forge.Entity.Generators is wired in as a build-time analyzer so all boilerplate is emitted at compile time with no runtime reflection.

Forge.Entity.Generators

Target framework
netstandard2.0
Namespace
Forge.Entity.Generators

Roslyn incremental source generator that processes every [Entity]-annotated partial class and emits the second partial half: IRI materialization, equality-by-IRI, property-change guards, and lazy-loading stubs. Runs entirely inside the compiler — zero runtime overhead — and is consumed by Forge.Entity (and by end-user projects) as an analyzer reference.

Forge.Entity.Repository

Target framework
net10.0
Namespace
Forge.Entity.Repository

Abstract persistence layer. Defines the store contract (IEntityStore, IEntityLoader, IRdfTripleSink), the typed application facade (IEntityRepository<T> / EntityRepository<T>), and the RDF mapping infrastructure (IRdfMapper, IRdfMapperRegistry, ReflectionRdfMapper, LiteralCodec, PredicateResolver). Acts as the shared boundary between backend implementations and the rest of the stack.

Forge.Entity.Sparql

Target framework
net10.0

Forge.Entity.Sparql

LINQ-to-SPARQL provider. Exposes store.Query<T>() returning an IOrderedQueryable<T> / IAsyncEnumerable<T> that translates expression trees into SPARQL SELECT queries via LinqToSparqlVisitor and SparqlEmitter, then executes them against any ISparqlQueryStore backend.

Forge.Entity.Repository.InMemory

Target framework
net10.0
Namespace
Forge.Entity.Repository.InMemory

In-memory IEntityStore and ISparqlQueryStore backed by a dotNetRDF Graph. Uses direct graph traversal for load/save/delete and dotNetRDF's in-process SPARQL engine for query execution; intended for unit tests and embedded scenarios, and shares a behavioral test suite with the GraphDB backend to guarantee interchangeability.

Forge.Entity.Repository.GraphDb

Target framework
net10.0
Namespace
Forge.Entity.Repository.GraphDb

HTTP-based IEntityStore and ISparqlQueryStore targeting the Ontotext GraphDB SPARQL endpoint. Reads via SPARQL CONSTRUCT, writes via SPARQL UPDATE (INSERT DATA / DELETE WHERE), and uses IHttpClientFactory with optional HTTP Basic authentication configured through GraphDbOptions.

Forge.Entity.Operations

Target framework
net10.0
Namespace
Forge.Entity.Operations

Ambient active-record bridge. EntityOperations.Use(store) binds an IEntityStore to the current async control flow so that generator-emitted instance methods (CreateAsync, ReadAsync, UpdateAsync, DeleteAsync) resolve the store without explicit injection. Orchestrates Forge.Entity.Repository and Forge.Entity.Sparql.

Forge.Entity.Operations.Generators

Target framework
netstandard2.0
Namespace
Forge.Entity.Operations.Generators

Roslyn incremental source generator that emits active-record CRUD methods (CreateAsync, ReadAsync, UpdateAsync, DeleteAsync) onto every [Entity]-annotated partial class in the consuming project. Generated file hints use the pattern {Namespace}.{TypeName}.g.ops.cs; consumed by end-user projects as an analyzer reference alongside Forge.Entity.Operations.
