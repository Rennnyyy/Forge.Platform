Forge.Platform — Projects

Forge.Platform is a .NET 10 library platform for building richly-typed, RDF-aware domain models. It delivers a full vertical stack: compile-time code generation via Roslyn source generators, IRI-based entity identity, an abstract RDF persistence contract with in-memory and Ontotext GraphDB backends, a LINQ-to-SPARQL query translation layer, ambient active-record operations, SHACL-based aspect validation, an authorization guard abstraction, a structured command/response capability dispatch pipeline, and cross-cutting execution correlation — all wired for ASP.NET Core through companion HTTP integration packages.

Forge.Entity.Generators

Target framework
netstandard2.0
Namespace
Forge.Entity.Generators

Roslyn incremental source generator (IsRoslynComponent=true) that processes every [Entity]-annotated partial class at compile time and emits the second partial half: IEntity implementation, IRI materialization, identity sealing, and lazy-loading stubs. Consumed by Forge.Entity (and by end-user projects) as a build-time analyzer reference — zero runtime overhead.

Forge.Entity

Target framework
net10.0
Namespace
Forge.Entity

Core runtime library for the entity model. Provides IEntity, EntityBase, the full annotation surface ([Entity], [Identity], [IdentityPart], [Predicate], [Owning], [Inverse]), IRI materialization, lazy-loading types (EntityRef<T>, EntityRefCollection<T>), and EntitySession. Forge.Entity.Generators is wired in as a build-time analyzer so all boilerplate is emitted at compile time with no runtime reflection.

Forge.Repository

Target framework

Forge.Repository

Abstract persistence layer and backend-agnostic contract. Defines IEntityStore, IEntityRepository<T>, ISparqlQueryStore, RDF primitives (RdfGraph, RdfTriple, RdfTerm), and the full RDF mapping infrastructure (IRdfMapper, IRdfMapperRegistry, ReflectionRdfMapper, LiteralCodec, PredicateResolver). Acts as the shared boundary between the two backend implementations and all higher-level consumers.

Forge.Sparql

LINQ-to-SPARQL query translation layer. Exposes SparqlQueryable<T> (implementing IOrderedQueryable<T> and IAsyncEnumerable<T>), which translates C# expression trees through LinqToSparqlVisitor and SparqlEmitter into SPARQL SELECT queries and executes them against any ISparqlQueryStore backend.

Forge.Repository.InMemory

In-memory IEntityStore and ISparqlQueryStore backed by a dotNetRDF Graph. Uses direct graph traversal for load/save/delete operations and dotNetRDF's in-process SPARQL engine for queries; designed for unit tests and embedded scenarios, and shares a behavioral test suite with the GraphDB backend to guarantee interchangeability.

Forge.Repository.GraphDb

HTTP-based IEntityStore and ISparqlQueryStore targeting the Ontotext GraphDB SPARQL endpoint. Reads via SPARQL CONSTRUCT, writes via SPARQL UPDATE (INSERT DATA / DELETE WHERE), and uses IHttpClientFactory with optional Basic authentication configured through GraphDbOptions.

Forge.Aspects

Full SHACL-based validation system for the platform. Implements AspectStore (thread-safe registration and resolution of operation, query, message, and capability aspects), SHACL shape parsing from Turtle TTL graphs (ShaclShapeGraph), TTL-based aspect loading, and enforcement wrappers (AspectEnforcingEntityStore, AspectEnforcingTransactionalStore) that validate entities and messages against registered SHACL shapes.

Forge.Authorization

Target framework
net10.0
Namespace
Forge.Authorization

Authorization primitive layer. Defines IAspectGuard — a single-method contract that decides whether a given agent token may act under a given aspect policy — and ships AllowAllAspectGuard as the default allow-everything stub. GuardedTransactionalStore wraps an IEntityStore to enforce the guard before every mutating operation.

Forge.Execution.Http

ASP.NET Core integration for the Execution context. Forge.Operations

Ambient active-record operations bridge. Forge.Operations.Generators

Forge.Capability

Target framework
net10.0
Namespace
Forge.Capability

Command/response capability dispatch pipeline. ICapabilityDispatcher<TCommand,TResponse> orchestrates a four-stage pipeline: resolve the CapabilityAspect from the store, validate the inbound command via SHACL, invoke the user-supplied ICapabilityHandler<TCommand,TResponse>, then validate the response and any emitted events. Integrates with both Forge.Aspects and Forge.Authorization.

Forge.Operations.Http

ASP.NET Core integration for entity operations. MapOperations() discovers all [OperationEndpoints]-annotated entities and registers a five-verb REST endpoint set (POST create, GET list/read, PUT update, DELETE delete) per entity. Integrates the execution correlation scope from Forge.Execution.Http into every request and handles JSON serialization of entity and relation types.
