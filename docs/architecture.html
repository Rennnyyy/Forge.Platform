Layer descriptions

Foundations Foundations

The lowest-level abstraction packages. Forge.Execution provides the ExecutionCorrelation ambient scope and error types. Forge.Aspects.Abstractions defines the IAspect contract and Aspect.NoOpIri sentinel. Forge.Messaging.Abstractions defines IMessageProducer, IMessageConsumer, and MessageEnvelope<T>. Forge.ObjectStorage.Abstractions defines IObjectStore. None of these depend on any other Forge slice.

Packages
Execution, Aspects.Abstractions, Messaging.Abstractions, ObjectStorage.Abstractions
Depends on
—

Entity & Persistence

Defines the entity programming model (IEntity, [Entity], [Predicate], [Identity]) and the RDF backed repository. Forge.Entity.Generators is a Roslyn source generator that emits structural partial classes. Forge.Repository defines IEntityStore, ITransactionalEntityStore, and EntityTransaction. Forge.Repository.InMemory and Forge.Repository.GraphDb are concrete backends. Forge.Sparql provides SPARQL query support.

Packages
Entity, Entity.Generators, Repository, Repository.InMemory, Repository.GraphDb, Sparql
Depends on
Foundations

Operations

Derived from the entity model, Forge.Operations exposes EntityOperations static CRUD helpers. Forge.Operations.Generators emits CreateAsync/ReadAsync/UpdateAsync/DeleteAsync/ListAsync active-record methods. Forge.Operations.Http scans assemblies for [OperationEndpoints] types and wires five REST endpoints per entity via MapOperations().

Packages
Operations, Operations.Generators, Operations.Http
Depends on
Entity & Persistence, HTTP Cross-Cutting

HTTP Cross-Cutting

Forge.Execution.Http threads ExecutionCorrelation through the HTTP pipeline, provides ExecutionEndpointHelper, and bridges BranchScope from the HTTP layer. It is the foundation dependency for all HTTP slices.

Packages
Execution.Http
Depends on
Foundations, Entity & Persistence, Branch

Authorization

Forge.Authorization provides IAspectGuard, GuardedTransactionalStore, and AuthorizationContext. Forge.Authorization.Http wires the HTTP startup guard and agent-token middleware. Forge.Aspects provides aspect enforcement, SHACL validation via dotNetRDF, and AddForgeAspects().

Packages
Authorization, Authorization.Http, Aspects
Depends on
Foundations, Entity & Persistence

Branch & Snapshot

Forge.Branch defines the Branch entity, manages named-graph isolation per branch, and exposes the BranchScope. Forge.Branch.Http wires five REST endpoints for branch CRUD and two endpoints for snapshot management via MapBranches() and MapSnapshots().

Packages
Branch, Branch.Http
Depends on
Entity & Persistence, HTTP Cross-Cutting, Authorization

Capability

Forge.Capability defines ICapabilityHandler<TReq,TResp>, [Capability], and the in-process dispatcher. Forge.Capability.Http exposes handlers over HTTP via MapCapabilities(). Forge.Capability.Messaging adds IAsyncCapabilityDispatcher for fire-and-forget and request-reply dispatch over the messaging layer (ADR-0022).

Packages
Capability, Capability.Http, Capability.Messaging
Depends on
Foundations, Authorization, HTTP Cross-Cutting

Messaging

Forge.Messaging.InMemory provides an in-process channel-based broker for testing. Forge.Messaging.Kafka provides a Confluent.Kafka implementation. Forge.Entity.Messaging wires the EventEmittingEntityStore decorator (ADR-0021) that publishes entity-change events to the history and state topics after each write.

Packages
Messaging.InMemory, Messaging.Kafka, Entity.Messaging
Depends on
Foundations (Messaging.Abstractions), Entity & Persistence

Object Storage

Forge.ObjectStorage.InMemory provides an in-memory blob store. Forge.ObjectStorage.Http registers [ObjectBearing] entities and wires eight REST routes per entity via MapObjectOperations() — covering metadata CRUD, streaming upload, binary download, and blob-only delete (ADR-0023).

Packages
ObjectStorage.InMemory, ObjectStorage.Http
Depends on
Foundations (ObjectStorage.Abstractions), Operations, HTTP Cross-Cutting
