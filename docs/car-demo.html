How the Conditions Work

Time effectivity (milestone eras)

TimeCondition is set on Usage edges. Edges with ValidFrom = 2026-01-01 are only traversed when the query carries a referenceDate ≥ 2026. The Race Edition Pack edge uses ValidTo = 2025-12-31 — it vanishes once the update-1 era begins. This is what "milestone" means in the product-structure world: a time window of applicability for a structural element, not a separate duplicate sub-tree.

Feature variants (drivetrain + trim)

EnumerationOptionCondition guards the variant branch edges (Powertrain → EV Package and Powertrain → ICE Package; the three interior edges). Both dimensions use open-world (IsRequired = false): omitting the dimension key from the query returns all variants simultaneously — the 150 % view for that axis.

Combined AND condition

The Interior → Luxury Interior edge carries a ConditionSet with two conditions: EnumerationOptionCondition(trim=luxury) AND TimeCondition(from 2026). Both must be satisfied simultaneously. A luxury configuration in the initial era (2025) still excludes this node.

Geometry layer

Each structural node (leaf or near-leaf) can own one or more Geometry nodes, connected via GeometryUsage edges. A GeometryUsage edge carries a 2D affine transformation matrix [a, b, c, d, e, f] and appears in two roles:

  • Structure → Geometry (S→G): the parent is a structural Node; the matrix positions the geometry on the shared assembly canvas (viewBox="0 0 560 230").
  • Geometry → Geometry (G→G): the parent is itself a Geometry node; the matrix positions the child relative to the parent’s local coordinate space, enabling hierarchical composition. In this demo the wheel pairs are sub-geometries of both car body outlines.

The geometry layer is queried separately from the structure tree (GET /api/entities/geometry-nodes and GET /api/entities/geometry-usages) and assembled client-side. Geometry nodes implement IStructure but are not returned by structure.configured-tree.get, which only traverses Usage edges. In the tree diagram above they are shown as diamond (◇) nodes to distinguish them from structural circles.
