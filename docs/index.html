Forge.Platform

A modular .NET 10 SDK for building RDF-backed domain APIs — from entity modelling to HTTP endpoints, messaging, branching, and binary storage.

What is Forge.Platform?

Forge.Platform is a layered .NET 10 SDK that treats RDF named graphs as the canonical persistence model for structured domain entities. It provides a coherent set of composable slices — each independently versionable — that cover the full lifecycle from compile-time code generation, through HTTP endpoint wiring, to asynchronous messaging and binary object storage. Consumers reference only the slices they need; the dependency injection layer assembles a correctly ordered decorator chain regardless of registration order.

The platform is designed around a single principle: domain logic should not know about persistence, transport, or policy. Entity types are plain annotated classes; Roslyn generators emit the boilerplate; the Repository and Operations layers handle persistence and HTTP mapping; Aspects and Authorization layers enforce SHACL and rule-based guards before any write reaches the store. Branching isolates work in named RDF graphs; Capability and Messaging layers provide synchronous and asynchronous dispatch patterns; Object Storage adds a first-class blob-management surface for binary-bearing entities.
