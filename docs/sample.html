Application.Sample

A runnable reference application that exercises every Forge.Platform capability: entity CRUD, branching & snapshots, capabilities, messaging, object storage, and aspects.

What It Demonstrates

Entity lifecycle

Eight entity types (Book, DataRecord, Artist, FeaturedArtist, Genre, Recording, Studio, TrackMaster) cover RDF-backed CRUD, type inheritance, enumeration entities, and object-bearing entities.

Operations HTTP layer

The source-generated [OperationEndpoints] attribute wires five REST endpoints per entity automatically — no controller code required.

Branching & Snapshots

All entity data is scoped to a named graph identified by the X-Forge-BranchIri header. Branches and immutable snapshots are first-class resources under api/branches and api/snapshots.

Capabilities (HTTP & Messaging)

Hand-crafted handlers (GreetHandler, CreateItemHandler, TriggerFaultHandler, AspectDemoHandler, …) are exposed synchronously via HTTP (MapCapabilities()) and asynchronously via an in-memory Kafka analogue (AddForgeCapabilityMessaging()).

Entity change events

EventEmittingEntityStore (wired by AddForgeEntityEvents()) publishes EntityCreated, EntityUpdated, and EntityDeleted events for Book to two in-memory consumers: an append-only EntityEventLog and a compacted EntityStateCache.

Object storage

TrackMaster is an [ObjectBearing] entity. Its binary asset (track master audio file) is managed via eight HTTP routes under api/objects/track-masters. An in-memory blob store is wired by AddForgeObjectStorageInMemory().

Program.cs Walkthrough

The sample DI and middleware wiring is annotated inline. The table below maps each registration call to its configuration reference:

var builder = WebApplication.CreateBuilder(args);

// ── 0. Messaging — in-memory broker + entity change events ───────────────────
// MUST be registered before AddForgeAspects() / AddForgeAuthorizationHttp()
// so EventEmittingEntityStore wins the unkeyed ITransactionalEntityStore slot.
// Chain: Guard → EventEmitting → AspectEnforcing → InMemoryBackend
builder.Services.AddForgeMessagingInMemory();
builder.Services.AddForgeEntityEvents();
builder.Services.AddForgeEntityMessaging<Book>(opts =>
{
    opts.TypeIri = "https://forge-it.net/entities/Book";
});

builder.Services.AddSingleton<EntityEventLog>();
builder.Services.AddSingleton<EntityStateCache>();
builder.Services.AddHostedService<BookHistoryConsumerService>();
builder.Services.AddHostedService<BookStateConsumerService>();

// ── 1. Aspect + authorization infrastructure ──────────────────────────────────
builder.Services.AddForgeAspects();
builder.Services.AddForgeAuthorizationHttp(builder.Configuration);

// ── 2. Entity repository ────────────────────────────────────────────────────── 
var repoBuilder = builder.Services.AddForgeEntityRepository(builder.Configuration);
if (string.Equals(
        builder.Configuration["Forge:EntityRepository:Backend"],
        "GraphDb", StringComparison.OrdinalIgnoreCase))
    repoBuilder.UseGraphDb();
else
    repoBuilder.UseInMemory();

// ── 3. Branch infrastructure ──────────────────────────────────────────────────
builder.Services.AddForgeBranchHttp(builder.Configuration);   // management store + aspects
builder.Services.AddBranchHttp(builder.Configuration);        // header provider

// ── 4-5. Capability handlers ──────────────────────────────────────────────────
builder.Services.AddCapabilityHandlersFromAssemblyContaining<Book>();
builder.Services.AddCapabilityHttp();
builder.Services.AddForgeCapabilityMessaging();

// ── 6. HTTP endpoint descriptors ──────────────────────────────────────────────
builder.Services.AddOperationEndpointsHttpFromAssemblyContaining<Book>();
builder.Services.AddForgeObjectStorageInMemory();
builder.Services.AddForgeObjectStorageHttpFromAssemblyContaining<TrackMaster>();

var app = builder.Build();

// ── Middleware ────────────────────────────────────────────────────────────────
app.UseBranchScope();       // X-Forge-BranchIri header → BranchScope.Current
app.Use(/* snapshot 422 handler */);

// ── Endpoints ─────────────────────────────────────────────────────────────────
app.MapCapabilities();        // api/capabilities/**
app.MapOperations();          // api/entities/{path}  (standard entities)
app.MapObjectOperations();    // api/entities/track-masters + api/objects/track-masters
app.MapBranches();            // api/branches
app.MapSnapshots();           // api/snapshots

app.MapGet("/api/diagnostics/entity-events", /* event log query */);

app.Run();

Entity Definitions

// Book — property-based plain identity, fully managed lifecycle
[Entity(Path = "books")]
[Identity(IdentityKind.PropertyBasedPlain)]
[OperationEndpoints]
public partial class Book
{
    public string Title  { get; set; } = "";
    public string Author { get; set; } = "";
    public int    Year   { get; set; }
}
// Artist — random UUID identity
[Entity(Path = "artists")]
[Identity(IdentityKind.Random)]
[OperationEndpoints]
public partial class Artist
{
    public string Name    { get; set; } = "";
    public string Country { get; set; } = "";
}

// FeaturedArtist — inherits Artist, overrides route path
[Entity]
[OperationEndpoints("featured-artists")]
public partial class FeaturedArtist : Artist
{
    public string Feature { get; set; } = "";
}
// Genre — enumeration entity (read-only via HTTP, no create/update/delete routes)
[Entity(Path = "genres")]
[Enumeration]
[Identity(IdentityKind.PropertyBasedPlain)]
public partial class Genre
{
    public string Label { get; set; } = "";
}
// TrackMaster — object-bearing entity (binary blob + metadata)
[ObjectBearing]
[OperationEndpoints]
public partial class TrackMaster
{
    public string Title    { get; set; } = "";
    public string Artist   { get; set; } = "";
    public int    Duration { get; set; }   // seconds
}

All entity classes are partial — the Forge.Entity.Generators Roslyn source generator adds RDF mapping code at compile time. The generator is referenced as an analyzer-style package reference in Application.Sample.csproj; no generated files are checked in.

Key Design Decisions

ADR-0019

Managed entity HTTP aspect enforcement: all HTTP entry-points (entity ops, branch ops) run through AspectEnforcingTransactionalStore. The sample wires aspects onto both the main store and the branch management store.

ADR-0021

Entity change event stream: EventEmittingEntityStore must be registered before the aspect/guard decorators so it sits above them in the decoration chain. The sample demonstrates the correct ordering in Program.cs step 0.

ADR-0022

Async capability command bus: AddForgeCapabilityMessaging() auto-discovers all [Capability] handlers and wires end-to-end async request/reply transport without per-handler configuration.

ADR-0023

Object storage slices: TrackMaster uses [ObjectBearing] to add an eight-route binary asset API layer on top of a standard entity. Object and entity lifecycles are managed independently.

ADR-0014

Registration-order-independent decorator chain: the deferred factory pattern ensures all decorators are applied regardless of the order AddForge…() calls appear in Program.cs.

ADR-0017

Shared operation contracts: all entities use the same shared Request/Response types from Forge.Operations.Http. Entity-specific types are only used for capability payloads.

Running the Sample

  1. Clone and restore 
    git clone <repo-url>
cd platform
dotnet restore
  2. Start with InMemory backend (default) 
    dotnet run --project samples/Application.Sample
  3. Optional: start with GraphDB backend 
    # Start GraphDB (Docker):
docker run -p 7200:7200 ontotext/graphdb:10.7.5

# Run with GraphDB environment (loads appsettings.GraphDb.json):
dotnet run --project samples/Application.Sample \
  --environment GraphDb
  4. Run Bruno API tests 
    bru run spec/forge --env local
  5. Run unit + integration tests 
    dotnet test tests/Application.Sample.Tests
